# Skill: Content Creation

## When to Load
Creating blog posts, LinkedIn posts, social threads, video scripts, newsletter content, or client-facing documentation about work being done.

## Workflow
1. Identify the key takeaway (one sentence: what should the reader walk away knowing?)
2. Identify the audience and format
3. Pull source material from wiki/, memory/, and raw/
4. Create outline for approval
5. Draft
6. Review against format-specific guidelines below
7. Final edit
8. Update content/_pipeline.md

## Format Guidelines

### Blog Post
- 800-2000 words
- Hook in first paragraph (problem or surprising insight)
- Headers every 2-3 paragraphs for scannability
- Code examples or screenshots where relevant
- End with a takeaway or next step, not a generic "conclusion"
- Personal voice, the author's perspective, not a textbook

### LinkedIn Post
- 1,300 characters max
- Hook in first line (stop the scroll)
- Personal angle or story
- 1-2 line breaks for readability
- End with a question or CTA for engagement
- 3-5 relevant hashtags

### Social Thread (X/Twitter)
- 5-10 tweets
- Each tweet must stand alone (someone might see just one)
- First tweet is the hook
- Last tweet is the CTA/summary
- Number the tweets (1/7, 2/7, etc.)

### Video Script
- Talking points format, not word-for-word script
- Note screen recording moments ("SHOW: the folder structure here")
- Keep each segment under 2 minutes
- Open with the problem, close with the result

### Notion Page (Client-Facing)
- Professional, clean layout
- Show process and thinking, not just deliverables
- Use Notion-native formatting (callouts, toggles, databases)
- Include relevant visuals or diagrams

## Rules
- Always also load @content/_pipeline.md for pipeline context
- Every piece of content traces back to a source (learning, decision, wiki page, or raw source)
- Update _pipeline.md when status changes
- Meta-document: the process of building things IS content. Always capture it.
