# Auto Memory Boundaries

Claude Code's native auto memory and Mind-Lint both stay enabled with clear separation.

## Native Auto Memory Handles
- Session-specific debugging notes
- Project-specific commands
- Ephemeral context useful for the next few sessions but not long-term

## Mind-Lint Handles
- Cross-project learnings
- Durable preferences and corrections
- Decisions with rationale
- Anything shareable or content-worthy
- Error rules from AI mistakes
- Wiki compilation and maintenance

## The Test
"Would this be useful in a completely different project?" If yes, Mind-Lint. If no, native auto memory.
