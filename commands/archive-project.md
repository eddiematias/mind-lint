# /archive-project — Final Knowledge Sweep

Wrap up a project with a comprehensive knowledge capture and crystallization.

## Steps
1. Ask which project is being archived
2. Capture any unlogged learnings, decisions, or corrections
3. Update decision statuses (active → completed where applicable)
4. Compile any unlogged knowledge into wiki via /compile

## Crystallization
5. Create a structured project digest:
   - Summary: what was built
   - Key learnings (with links to memory/learnings/ entries)
   - Decisions made (with links to memory/decisions/ entries)
   - Entities involved (people, tools, technologies)
   - Metrics (timeline, scope, outcome)
   - Content opportunities
6. Write digest to wiki/ as wiki/project-[name].md
7. Update wiki/_index.md
8. Create content idea in content/ideas/ titled "Build Story: [project name]"
9. Update content/_pipeline.md
10. Move project-specific files to archive/completed-projects/ if applicable
11. Report what was captured and archived
